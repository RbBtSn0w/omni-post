### 用户场景 1 - 安全的文件与 Cookie 管理 (优先级: P1)
用户希望能够上传和下载视频文件及账号 Cookie，而不会面临路径穿越漏洞导致的敏感数据外泄风险。
- **独立测试**: 尝试通过 `/downloadCookie?filePath=../../package.json` 下载文件，应被拦截并返回 400。

### 用户场景 2 - 可靠的多平台发布 (优先级: P1)
用户期望在启动发布任务时，系统能通过“账号锁”防止同一个账号被多个任务同时操作，并能优化浏览器资源占用。
- **独立测试**: 同时为账号 A 开启两个任务，第二个任务应返回 423。

### 用户场景 3 - 一致的账号状态 (优先级: P2)
用户希望在仪表盘看到准确的实时状态。
- **独立测试**: 模拟 Cookie 过期，调用 `/getValidAccounts` 应触发实时 Playwright 校验并更新数据库。

### 用户场景 4 - 自动化质量保障 (优先级: P2)
开发者提交代码后，CI 能自动拦截 Lint 错误（如缺失 .js 后缀）和安全漏洞。
- **独立测试**: 提交包含类型错误的代码，GitHub Actions 应报错并禁止合并。

## 功能需求 *(必填)*

### 核心功能需求 (对应 PR #142 CR 问题)

- **FR-001: 路径穿越防护**: 所有涉及文件读取（`/getFile`, `/download/:filename`）和 Cookie 下载的接口，必须使用 `path.resolve` 校验路径是否在允许的根目录内。
- **FR-002: ESM 稳定性**: 所有内部 TypeScript/JavaScript 导入语句必须显式包含 `.js` 后缀，防止 Node.js 运行时出现 `ERR_MODULE_NOT_FOUND`。
- **FR-003: 账号锁定强制执行**: `PublishExecutor` 必须在执行前获取 `LockManager` 锁，并在 `finally` 块中释放。
- **FR-004: 实时 Cookie 验证**: 账号状态接口必须在上次校验超过 3 小时时触发实时验证，且支持 `force=true` 参数跳过冷却。
- **FR-005: 资源自动清理**: SSE 登录流必须在任务结束或客户端断开时显式调用 `res.end()` 并移除所有事件监听器。
- **FR-006: Playwright 上下文优化**: 上传器（特别是 Bilibili）必须复用传入的 `BrowserContext`，禁止在内部重复创建浏览器实例。
- **FR-007: 响应结构对等**: 所有 API 响应必须严格遵循 `{ code: number, msg: string, data: any }` 结构。
- **FR-008: SQLite 测试隔离**: Vitest 环境清理必须递归删除所有辅助文件（-wal, -shm）。
- **FR-009: 自动化 CI 管道**: 在 `.github/workflows/` 下配置完整的 GitHub Actions，包含 Lint、TSC 检查、Vitest 及 CodeQL。
